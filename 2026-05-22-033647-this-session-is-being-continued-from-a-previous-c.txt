● ---
  框架全景

  这是什么

  PAPO（Perception-Augmented Policy Optimization）— 在 GRPO/DAPO
  基础上加了视觉感知增强的 RL 训练框架。底层是 VERL（Ray 分布式
  RL），专门训练多模态 VLM（视觉语言模型）。

  任务目标：训 Lingshu-7B（Qwen2.5-VL
  医学微调版）做医学证据定位——给图像+问题，模型输出 bounding box
  标出图中证据区域。

  ---
  训练流程（每个 step 做4件事）

  ① Rollout      模型对每个 prompt 生成 n=5 个回答（用 vLLM 加速推理）
  ② Reward       对每个回答打分（调用 reward_function）
  ③ Advantage    GRPO/DAPO 计算优势函数（组内对比，不需要 critic）
  ④ Actor Update FSDP 反向传播更新模型权重

  ---
  Reward Function（当前用的）

  路径：examples/reward_function/medical_evidence.py:compute_score

  模型输出格式要求：
  <evidence>{"boxes": [{"bbox": [x1,y1,x2,y2]}, ...], "num_boxes": N}</evidence>
  坐标范围 0–1000（归一化）。

  打分逻辑：

  ┌────────────────────────────────────────────┬────────────────────────────┐
  │                    情况                    │         overall 分         │
  ├────────────────────────────────────────────┼────────────────────────────┤
  │ 解析失败（格式错）                         │ -0.2                       │
  ├────────────────────────────────────────────┼────────────────────────────┤
  │ 解析成功：0.9 × matched_IoU + 0.1 × num_ok │ 0–1.0                      │
  ├────────────────────────────────────────────┼────────────────────────────┤
  │ evidence_deleted                           │ +0.5                       │
  │ 样本（应输出空框）预测为空                 │                            │
  ├────────────────────────────────────────────┼────────────────────────────┤
  │ evidence_deleted 但预测有框                │ -0.1 ~                     │
  │                                            │ -0.5（越重叠越惩罚）       │
  └────────────────────────────────────────────┴────────────────────────────┘

  ---
  Config 关键参数

  # ── 算法 ──────────────────────────────────────────────
  algorithm:
    adv_estimator: dapo        # grpo 或 dapo（dapo 自动过滤 all-same-reward 
  的组）
    disable_kl: true           # 不用 reference model 的 KL 惩罚
    use_kl_loss: true          # 但用 low_var_kl 稳定训练
    online_filtering: true     # DAPO: 过滤掉全对/全错的 batch（无学习信号）

    # PAPO 独有：感知增强 KL
    use_kl_prcp: true          # 开启感知 KL（核心创新点）
    kl_prcp_coef: 0.01         # 感知 KL 权重
    aug_config:

      black_prob: 0.6          # 60% 概率随机遮掉图像 patch

  # ── Actor 训练 ────────────────────────────────────────

  worker.actor.model.model_path: ...       # 初始权重路径

  ---
  PAPO vs 普通 GRPO 区别

  普通 GRPO 只靠 reward 信号更新。PAPO 额外加了一项损失：

  L_total = L_grpo + λ × KL(原图表示 || 遮掉patch后的表示)

  直觉：强迫模型理解图像局部内容，而不是靠语言模式猜答案。black_prob=0.6
  意味着每次前向传播随机遮掉 60% 的 14×14 patch，让模型不能依赖全图。

  1. DAPO 是什么，在哪里定义？

  DAPO = Dynamic sAmpling Policy Optimization，是 GRPO 的改进版。

  核心区别：

  ┌──────┬──────────────────────┬────────────────────────────────────────────┐
  │      │         GRPO         │                    DAPO                    │
  ├──────┼──────────────────────┼────────────────────────────────────────────┤
  │ 过滤 │ 不过滤               │ 过滤全对/全错的组（online_filtering=true） │
  ├──────┼──────────────────────┼────────────────────────────────────────────┤
  │ 原理 │ 组内所有样本参与更新 │ 全对或全错的组                             │
  │      │                      │ advantage=0，没有学习信号，跳过            │
  └──────┴──────────────────────┴────────────────────────────────────────────┘

  定义位置：verl/trainer/core_algos.py:213 — compute_dapo_outcome_advantage()

  这是 VERL 原始框架自带的，不是这个项目加的。

  ---
  2. KL 是什么

  KL = KL 散度（Kullback-Leibler divergence），衡量两个概率分布之间的差异。

  RL 训练中有两处 KL：

  ① 普通 KL（disable_kl: true，这里关了）
     KL(当前policy || 参考policy) → 防止模型训崩、偏离太远

  ② 感知 KL（use_kl_prcp: true，PAPO 独有）
     KL(原图logits || 遮掩图logits) → 下面解释

  ---
  3. 60% 随机遮掩 Patch 是什么，在哪里

  这是 PAPO 的核心创新，不是任务自带的。

  定义位置：verl/trainer/papo_utils.py:17 — random_patch_blackening()

  调用位置：verl/trainer/ray_trainer.py:553 — _aug_img_for_kl_prcp()

  做的事情：

  原图 ──────────────────────→ 模型 → logits_原图
  原图 → 随机遮掉60%的14×14块 → 模型 → logits_遮掩图

  损失加一项：KL(logits_原图 || logits_遮掩图)

  目的：强迫模型对局部遮掩不敏感，学会理解图像的整体语义，而不是靠某个局部细节猜
  答案。这对医学图像定位特别重要——不能靠图像边缘或背景走捷径。

  ---
  4. PAPO 是什么（完整）

  PAPO = Perception-Augmented Policy Optimization

  = GRPO/DAPO（文字RL）+ 感知增强（图像理解强化）

  完整损失函数：

  L = L_GRPO/DAPO            ← 基础 RL（靠 reward 优化）
    + λ₁ × KL_感知           ← 遮掩图对比（PAPO 新增）
    + λ₂ × 熵正则(可选)      ← use_aug_entropy_loss/use_ori_entropy_loss

  这个项目就是用 PAPO 训练 Lingshu-7B 做医学证据框定位任务。

● ---
  vLLM Rollout Worker 的角色

  RL 训练最慢的环节是生成（rollout）——模型对每个 prompt 要采样 5
  个回答，自回归逐 token 生成，极慢。

  没有 vLLM：PyTorch 逐 token forward → 很慢
  有 vLLM：   PagedAttention + KV Cache 复用 → 快 3-10x

  框架把角色分离：
  Actor（FSDP）  ← 负责梯度更新，用 PyTorch
  Rollout（vLLM）← 负责生成回答，用 vLLM
  每次 update 后，Actor 权重同步给 vLLM worker，再生成下一批。sharding_manager/
  就是做这个权重同步的。
