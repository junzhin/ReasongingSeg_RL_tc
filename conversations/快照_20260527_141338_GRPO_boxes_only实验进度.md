### 2.3 Reward Function 关键设计（boxes-only 版）

```
情况1：格式解析失败（无<evidence>标签/JSON错/坐标非法/num_boxes不一致）→ overall = -0.2
情况1.5：gt_num=0（脏数据）→ overall = -0.2 + RuntimeWarning
情况2：gt有框但pred空（漏检）→ overall = -0.2
情况3：pred有框 + gt有框 → overall = matched_IoU（贪心一对一匹配，取平均）
```

删掉了原版的 `0.1 × num_ok` 权重，overall = 纯 IoU。

## 3. 当前断点

### 正在进行
- **Exp01（GRPO boxes-only）正在运行中**，已到 global_step 62
- TensorBoard 观察到：`parse_ok_reward ≈ 0.12`（震荡下降），`overall_reward = -0.2`（卡死），`bbox_iou = 0`

### 卡在的具体问题
**GRPO 冷启动困境**：

Lingshu-7B 从未见过 `<evidence>` 格式，初始 parse_ok 只有 ~6-12%。GRPO 的 advantage 计算为：
```
advantage = (reward - mean) / std
```
当一个 prompt 的 5 个 rollout 全部格式失败 → 全是 -0.2 → std=0 → advantage=0 → 梯度为零 → 模型无法学习格式。

**当前讨论结论**：
- Reward 分级（给有标签但 JSON 错的 -0.05 而非 -0.2）**可能不够**，因为模型根本不知道 `<evidence>` 标签的存在
- **必须先做 SFT 热身**，让模型学会稳定输出格式，再切换到 GRPO
- 问题：是否有现成的已做过 SFT 能输出 `<evidence>` 格式的模型权重？

## 4. 避坑档案

### 坑1：math_perception.jinja 与医学 reward 格式冲突
- **现象**：overall 一直 -0.2，验证输出为 `\boxed{5}`
- **原因**：`math_perception.jinja` 末尾强制加 `\boxed{}` 指令，与 `<evidence>` 格式完全冲突
- **结论**：医学任务必须用 `medical_evidence.jinja`（只有 `{{ content | trim }}`）

### 坑2：evidence_deleted 样本导致 reward hacking
- **现象**：anti_ray reward 快速到 1，bbox_iou = 0
- **原因**：空框得 +0.5，是所有策略中最容易执行的，模型对所有输入输出空框
- **结论**：boxes-only 实验必须过滤掉 evidence_deleted 样本

### 坑3：原版 filter 只靠 sample_type 标签
- **风险**：original/background 样本里可能有 num_boxes=0 的脏数据
- **解法**：改为双保险（标签层 + 内容层：num_boxes>0 且至少一个坐标合法的框）

### 坑4：GRPO 冷启动（当前核心问题）
- **现象**：parse_ok 震荡下降，overall 锁死 -0.2，bbox_iou 永远 0
- **原因**：模型不认识格式 → 所有 rollout 失败 → std=0 → 梯度为零 → 模型随机漂移
- **结论**：纯 reward 分级可能不够，需要 SFT 热身
